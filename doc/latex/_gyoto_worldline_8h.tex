Define the wrapper accessors used in \mbox{\hyperlink{_gyoto_worldline_8h_aa765beed747e6fb92152b6188097dba7}{GYOTO\+\_\+\+WORLDLINE\+\_\+\+PROPERTIES(class)}} 

This macro, which is called automatically by \mbox{\hyperlink{_gyoto_worldline_8h_a1be18e74aa4c1176ff9bcb6e72cc3faf}{GYOTO\+\_\+\+WORLDLINE\+\_\+\+PROPERTY\+\_\+\+END(c, a)}}, must be called once with the definition of the methods (.C file) of any class that derives from Worldline. The corresponding macro GYOTO\+\_\+\+WORLDLINE must be called in the corresponding class declaration (.h file).

This is made necessary by how multiple inheritence works\+: directly using the accessors in the Worldline API leads to segfault at runtime (unless too much extra care is taken) and may go unnoticed.

These accessors must be declared in the class declaration using the GYOTO\+\_\+\+WORLDLINE macro. \mbox{\Hypertarget{_gyoto_worldline_8h_aa765beed747e6fb92152b6188097dba7}\label{_gyoto_worldline_8h_aa765beed747e6fb92152b6188097dba7}}
