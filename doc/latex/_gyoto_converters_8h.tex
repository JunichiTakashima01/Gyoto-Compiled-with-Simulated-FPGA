\doxysubsection{Detailed Description}
GYOTO converters. 

As a (recommended) option, \mbox{\hyperlink{namespace_gyoto}{Gyoto}} can use the libudunits2 library by UCAR/\+Unidata (\href{http://www.unidata.ucar.edu/software/udunits/udunits-2/udunits2.html}{\texttt{ http\+://www.\+unidata.\+ucar.\+edu/software/udunits/udunits-\/2/udunits2.\+html}}) to perform conversions to and from various units. Since udunits parses units, the following are equivalent for instance\+: \char`\"{}m\+Jy.\+sr-\/1\char`\"{}, \char`\"{}m\+Jy/sr\char`\"{}, \char`\"{}1e-\/3\+Jy/sr\char`\"{}. \mbox{\hyperlink{namespace_gyoto}{Gyoto}} considers every string as UTF-\/8\+: either use UTF-\/8 or stay with ASCII. This means \char`\"{}µ\char`\"{} can be used for \char`\"{}micro\char`\"{} and powers can be noted as superscripts\+: m³, pix².

In addition to the stock units known by udunits2, \mbox{\hyperlink{namespace_gyoto}{Gyoto}} registers the following (which may be used in any context)\+: Jansky (symbol Jy), symbol \char`\"{}pc\char`\"{} for parsec, \char`\"{}sunradius\char`\"{}, \char`\"{}sunmass\char`\"{}, symbol \char`\"{}as\char`\"{} for \char`\"{}arcsec\char`\"{}.

Other units are context-\/sensitive\+: \char`\"{}geometrical\char`\"{} allows converting between geometrical units and other legnth units, but only when a Metric is defined, and may not be used (yet) in compound units. Likewise, \char`\"{}geometrical\+\_\+time\char`\"{} can be used as duration unit whenever a Metric is defined, but not in a compound unit.

When a Screen is defined, \char`\"{}pix\char`\"{} can be used as an angle unit (you need to call Screen\+::map\+Pix\+Unit() and Screen\+::unmap\+Screen\+Unit(), which is done automatically in certain contexts).

Units can often be specified using the \char`\"{}unit\char`\"{} XML attribute\+:

\DoxyCodeLine{   </Mass>}

\end{DoxyCode}
