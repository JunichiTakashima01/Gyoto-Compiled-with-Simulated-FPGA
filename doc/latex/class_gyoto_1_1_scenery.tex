\doxysubsection{Detailed Description}
Ray-\/tracing scene. 

An \mbox{\hyperlink{class_gyoto_1_1_scenery}{Scenery}} contains\+:
\begin{DoxyItemize}
\item a \mbox{\hyperlink{namespace_gyoto_1_1_metric}{Metric}}\+: used in \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}}, \mbox{\hyperlink{class_gyoto_1_1_screen}{Screen}} and \mbox{\hyperlink{class_gyoto_1_1_photon}{Photon}};
\item a \mbox{\hyperlink{class_gyoto_1_1_screen}{Screen}}\+: sets the field-\/of-\/view, the position of the camera, the observation time, and the \mbox{\hyperlink{namespace_gyoto_1_1_spectrometer}{Spectrometer}};
\item an \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}}\+: light emitter.
\end{DoxyItemize}

In addition, Quantities may be specified (or the default Quantity will be produced\+: generally Intensity). Not all \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}} implement all Quantities. The order in which Quantities are listed is not relevant (it is not stored). Possible Quantities\+:


\begin{DoxyItemize}
\item Intensity\+: the intensity that reaches the object, integrated over the line-\/of-\/sight;
\item Emission\+Time\+: date of emission;
\item Min\+Distance\+: minimum distance between the \mbox{\hyperlink{class_gyoto_1_1_photon}{Photon}} reaching each pixel and the \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}};
\item First\+Dist\+Min\+: last closest approach between \mbox{\hyperlink{class_gyoto_1_1_photon}{Photon}} and \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}};
\item Redshift;
\item Impact\+Coords\+: 8-\/coordinates of the object and photon at impact;
\item \mbox{\hyperlink{namespace_gyoto_1_1_spectrum}{Spectrum}}\+: I\textsubscript{{$\nu$}} computed at various values frequencies, corresponding to the \mbox{\hyperlink{class_gyoto_1_1_screen}{Screen}}\textquotesingle{}s \mbox{\hyperlink{namespace_gyoto_1_1_spectrometer}{Spectrometer}}.
\item Bin\+Spectrum\+: {$\int$}\textsubscript{{$\nu$}\textsubscript{1}}\textsuperscript{{$\nu$}\textsubscript{2}}I\textsubscript{{$\nu$}}d{$\nu$} computed between various ({$\nu$}\textsubscript{1}, {$\nu$}\textsubscript{2} pairs corresponding to the \mbox{\hyperlink{class_gyoto_1_1_screen}{Screen}}\textquotesingle{}s \mbox{\hyperlink{namespace_gyoto_1_1_spectrometer}{Spectrometer}}. This is what a physical spectrometer measures.
\end{DoxyItemize}

In addition, it is possible to ray-\/trace an image using several cores on a single machine (if \mbox{\hyperlink{namespace_gyoto}{Gyoto}} has been compiled with POSIX threads support). The number of threads can be specified using NThreads entity. Setting NThreads to 0 is equivalent to setting it to 1. Beware that setting NThreads to a number higher than the actual number of cores available on the machine usually leads to a decrease in performance.

Finally, \mbox{\hyperlink{class_gyoto_1_1_scenery}{Scenery}} accepts a number of numerical tuning parameters that are passed directly to the underlying photons (actually, the \mbox{\hyperlink{class_gyoto_1_1_scenery}{Scenery}} object holds a \mbox{\hyperlink{class_gyoto_1_1_photon}{Photon}} instance which stores many parameters, including the \mbox{\hyperlink{namespace_gyoto_1_1_metric}{Metric}} and \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}})\+: Adaptive/\+Non\+Adaptive, Delta, Minimum\+Time, Max\+Iter, Primary\+Only.

A subcontractor builds an object upon order from the \mbox{\hyperlink{class_gyoto_1_1_factory}{Factory}}. 

Various classes need to provide a subcontractor to be able to instantiate themselves upon order from the \mbox{\hyperlink{class_gyoto_1_1_factory}{Factory}}. A subcontractor is a function (often a static member function) which accepts a pointer to a \mbox{\hyperlink{class_gyoto_1_1_factory_messenger}{Factory\+Messenger}} as unique parameter, communicates with the \mbox{\hyperlink{class_gyoto_1_1_factory}{Factory}} using this messenger to read an XML description of the object to build, and returns this objet. \mbox{\hyperlink{class_gyoto_1_1_smart_pointee_afe17bea3c33e2187940cfdd5543c1d14}{Smart\+Pointee\+::\+Subcontractor\+\_\+t}}$\ast$ is just generic enough a typedef to cast to and from other subcontractor types\+: \mbox{\hyperlink{namespace_gyoto_1_1_astrobj_aa53c7ada58c8c8f3799c3485b7d8f3bb}{Astrobj\+::\+Subcontractor\+\_\+t}}, \mbox{\hyperlink{namespace_gyoto_1_1_metric_ad6b33a642cc94ad67855184d2adb95df}{Metric\+::\+Subcontractor\+\_\+t}}, \mbox{\hyperlink{namespace_gyoto_1_1_spectrum_a163055eef14aee22dd8fa5188812d889}{Spectrum\+::\+Subcontractor\+\_\+t}}. A subcontractor needs to be registered using the relevant \mbox{\hyperlink{namespace_gyoto_1_1_astrobj_abb6caf3023a51cb77b5f6793ac03b188}{Register()}} function\+: \mbox{\hyperlink{namespace_gyoto_1_1_astrobj_abb6caf3023a51cb77b5f6793ac03b188}{Astrobj\+::\+Register()}}, \mbox{\hyperlink{namespace_gyoto_1_1_metric_a602cba49b001e7470f3d439d1ab3e8f7}{Metric\+::\+Register()}}, \mbox{\hyperlink{namespace_gyoto_1_1_spectrum_a517874749d838d7121880e7d9d14c626}{Spectrum\+::\+Register()}}.
