\doxysubsection{Detailed Description}
The camera with which the \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}} is observed. 

In the observer-\/centric point-\/of-\/view, the center of the \mbox{\hyperlink{namespace_gyoto_1_1_metric}{Metric}}\textquotesingle{}s coordinate system is positioned relatively to the observing \mbox{\hyperlink{class_gyoto_1_1_screen}{Screen}} using three Euler angles and the distance (in meters). The three Euler angles are\+:
\begin{DoxyItemize}
\item position angle of the line of nodes (North of East);
\item inclination (0 = face-\/on);
\item argument of the X axis of the \mbox{\hyperlink{namespace_gyoto_1_1_metric}{Metric}}\textquotesingle{}s coordinate system. We use the z-\/x-\/z convention. See \href{http://en.wikipedia.org/wiki/Euler_angles}{\texttt{ http\+://en.\+wikipedia.\+org/wiki/\+Euler\+\_\+angles}}
\end{DoxyItemize}

In addition, the \mbox{\hyperlink{class_gyoto_1_1_screen}{Screen}} conveys\+:
\begin{DoxyItemize}
\item the observing date (in geometrical units, but expect it to change to seconds in a future version);
\item the field-\/of-\/view of the image;
\item the resolution of the camera\+: number of pixels on each side (the camera is square);
\item the observing frequency.
\end{DoxyItemize}

The scalar Freq\+Obs defines the observing frequency for \mbox{\hyperlink{class_gyoto_1_1_scenery}{Scenery}} quantity Intensity.

Likewise, a \mbox{\hyperlink{namespace_gyoto_1_1_spectrometer}{Gyoto\+::\+Spectrometer}} defines for which frequencies spectra are computed (when the Quantity \mbox{\hyperlink{namespace_gyoto_1_1_spectrum}{Spectrum}} is requested in the \mbox{\hyperlink{class_gyoto_1_1_scenery}{Scenery}}).

For the sake of theoreticians, there is an alternate way of specifying the relative position of the \mbox{\hyperlink{class_gyoto_1_1_screen}{Screen}} and \mbox{\hyperlink{namespace_gyoto_1_1_metric}{Metric}}, by specifying the 4-\/coordinates of the \mbox{\hyperlink{class_gyoto_1_1_screen}{Screen}} in the \mbox{\hyperlink{namespace_gyoto_1_1_metric}{Metric}}\textquotesingle{}s coordinate system (in that case, eerything is specified in geometrical units).

\mbox{\hyperlink{namespace_gyoto_1_1_units}{Units}} can be specified using the unit attribute in the XML file, for instance\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{<Distance unit=\textcolor{stringliteral}{"{}kpc"{}}> 8 </Distance>}

\end{DoxyCode}


Possible units are (with \mbox{[}\mbox{]} noting the default)\+:
\begin{DoxyItemize}
\item distance\+: \mbox{[}m\mbox{]}, geometrical, cm, km, AU, ly, pc, kpc, Mpc;
\item PALN, inclination, argument\+: \mbox{[}rad\mbox{]}, deg.
\item frequency\+: \mbox{[}Hz\mbox{]}, m, GeV...
\end{DoxyItemize}

When the distance is really large and most of the ray-\/tracing would happen de facto in flat space, the camera is transported to a location at a reasonable distance from the metric and the images are scaled accordingly. The default value for this distance should be fine, but it can be customized using the \char`\"{}dmax\char`\"{} attribute of the \char`\"{}\+Distance\char`\"{} element. \char`\"{}dmax\char`\"{} is always expressed in geometrical units\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{<Distance unit=\textcolor{stringliteral}{"{}kpc"{}} dmax=\textcolor{stringliteral}{"{}1e7"{}}> 8 </Distance>}

\end{DoxyCode}


Symptoms when dmax is too large include pixelization of the image (neighbouring photons are numerically identical) and other numerical overflows. dmax is too small when it is apparent that changing it yields projection effects. dmax must be large compared to rmax in the \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}} and ideally, changing it by an order of magnitude should not yield significant changes in the ray-\/traced image.

A mask may be used to limit ray-\/tracing to only some portions of the field. The \mbox{\hyperlink{class_gyoto_1_1_scenery}{Scenery}} checks whether a mask is to be used using \mbox{\hyperlink{class_gyoto_1_1_screen_ab6a2c480414850b30659855936493140}{Screen\+::operator()(size\+\_\+t i, size\+\_\+t j)}}. The mask can be loaded from a FITS file as a square image of doubles\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{<Mask>maskfile.fits</Mask>}

\end{DoxyCode}
 The mask needs to be have the same size as the \mbox{\hyperlink{class_gyoto_1_1_screen}{Screen}} itself, so loading a mask also sets the resolution, and changing the resolution after setting a mask also removes the mask. The content of the Mask entity is parsed by \mbox{\hyperlink{class_gyoto_1_1_factory_a1adf0078f2541926fcfeac0221a21206}{Factory\+::full\+Path()}}, so it can be an absolute path, a path relative to where the XML file is stored, or relative to the current working directory if prefixed with \char`\"{}\`{}pwd\`{}/\char`\"{}.

Return True if this object is thread-\/safe, i.\+e. if an instance and its clone can be used in parallel threads (in the context of Scenery\+::raytrace()). Known objects which are not thread-\/safe include Lorene metrics and everything from the \mbox{\hyperlink{namespace_gyoto_1_1_python}{Python}} plug-\/in.
