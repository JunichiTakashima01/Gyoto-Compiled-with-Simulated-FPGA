\doxysubsection{Detailed Description}
Access to astronomical objects. 

Objects which are supposed to be the target of the ray-\/tracing code should inherit from the \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic}{Gyoto\+::\+Astrobj\+::\+Generic}} class.

When implementing a new object, you must\+:
\begin{DoxyItemize}
\item make sure the object can be loaded from XML by providing a Subcontractor\+\_\+t using the \mbox{\hyperlink{namespace_gyoto_1_1_astrobj_abb6caf3023a51cb77b5f6793ac03b188}{Gyoto\+::\+Astrobj\+::\+Register}}(std\+::string name, \mbox{\hyperlink{namespace_gyoto_1_1_astrobj_aa53c7ada58c8c8f3799c3485b7d8f3bb}{Gyoto\+::\+Astrobj\+::\+Subcontractor\+\_\+t}}$\ast$ scp) function;
\item make sure this subcontractor is registerred in the initialization routine of your plug-\/in;
\item make sure \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic_a73a731c15af0c247fc4c46ca934b1b14}{Generic\+::\+Impact()}} works (see below).
\end{DoxyItemize}

In addition, you should make sure that your object plays nicely in the Yorick plug-\/in, which means\+:
\begin{DoxyItemize}
\item implement the copy constructor and the \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic_af5e9f81e6888919dea2437639a871a3e}{Generic\+::clone()}} method;
\item implement the fill\+Element method, used for printing and saving to XML.
\end{DoxyItemize}

There are basically two ways of making \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic_a73a731c15af0c247fc4c46ca934b1b14}{Generic\+::\+Impact()}} work\+: either by making the \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}} a sub-\/class of the low-\/level \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic}{Gyoto\+::\+Astrobj\+::\+Generic}} class ans providing your own \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic_a73a731c15af0c247fc4c46ca934b1b14}{Generic\+::\+Impact()}} function (which, in principle, should rely on \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic_aa6f20e6d108357f6355493c4fdd5bb2f}{Generic\+::process\+Hit\+Quantities()}}), or by making the \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}} a sub-\/class of the higher-\/level \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_standard}{Gyoto\+::\+Astrobj\+::\+Standard}} class and implementing two lower level, simpler functions which are used by the \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_standard_a1d1cc899a7919c4369261a83c7656ad7}{Standard\+::\+Impact()}}\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_standard_a764b9a17fee369e61349acd3aa4ddd88}{Standard\+::operator()()}} yields a distance or potential defining the interior of the object;
\item \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_standard_acdc2e44eb25269f41b0710f62acfee40}{Standard\+::get\+Velocity()}} yields the velocity field of the fluid.
\end{DoxyItemize}

\mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic_aa6f20e6d108357f6355493c4fdd5bb2f}{Generic\+::process\+Hit\+Quantities()}} itself is an intermediate-\/level function which you may choose to reimplement. It uses three low-\/level, easy to implement functions\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic_a93d130767671c25823d5ec3fd477e28a}{Generic\+::emission()}};
\item \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic_a320317a0c347ab4317152b93952dab24}{Generic\+::integrate\+Emission()}};
\item \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic_a12691b14048aa4e3aa9fad7a427a198d}{Generic\+::transmission()}}. Default implementations of these three functions exist, they have little physical relevance but allow quick 0-\/th order vizualisation of your object.
\end{DoxyItemize}

To be usable, a \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic}{Astrobj\+::\+Generic}} (or \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_standard}{Astrobj\+::\+Standard}}) sub-\/classe should register an \mbox{\hyperlink{namespace_gyoto_1_1_astrobj_aa53c7ada58c8c8f3799c3485b7d8f3bb}{Astrobj\+::\+Subcontractor\+\_\+t}} function using the \mbox{\hyperlink{namespace_gyoto_1_1_astrobj_abb6caf3023a51cb77b5f6793ac03b188}{Astrobj\+::\+Register()}} function. See also \mbox{\hyperlink{writing_plugins_page}{Writing plug-\/ins for Gyoto}} . If your clas implements set\+Parameter() and/or, if necessary, set\+Parameters(), registering it is normally done using the provided template\+: 
\begin{DoxyCode}{0}

\end{DoxyCode}

Query the \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}} register. 

Query the \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}} register to get the \mbox{\hyperlink{namespace_gyoto_1_1_astrobj_aa53c7ada58c8c8f3799c3485b7d8f3bb}{Astrobj\+::\+Subcontractor\+\_\+t}} corresponding to a given kind name.
