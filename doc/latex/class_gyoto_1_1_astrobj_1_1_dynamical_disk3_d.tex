\doxysubsection{Detailed Description}
Geometrically thick optically thin disk read from a set of FITS files. 

This class describes a \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_pattern_disk_b_b}{Pattern\+Disk\+BB}} that evolves dynamically. It is described by a set of FITS files for different times. Its emission is blackbody.

The disk is assumed to be described by a regular, constant in time, grid.

The metric must be Kerr in BL coordinates.

\mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_disk3_d_a88f58b8244ee5ec4cdade4ba38cba23b}{Impact()}} checks whether a \mbox{\hyperlink{class_gyoto_1_1_photon}{Photon}} impacts the object between two integration steps of the photon\textquotesingle{}s trajectory (those two steps are photon-\/$>$get\+Coord(index, coord1) and photon-\/$>$get\+Coord(index+1, coord2)). Impact returns 1 if the photon impacts the object between these two steps, else 0. In many cases of geometrically thick obects, the implementation \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_standard_a1d1cc899a7919c4369261a83c7656ad7}{Astrobj\+::\+Standard\+::\+Impact()}} will be fine.

Impact will call \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic_aa6f20e6d108357f6355493c4fdd5bb2f}{Generic\+::process\+Hit\+Quantities()}} (which is virtual and may be re-\/implemented) to compute observable properties on demand\+: if the data pointer is non-\/\+NULL, the object will look in it for pointers to properties which apply to its kind. If a pointer to a property known to this object is present, then the property is computed and store at the pointed-\/to address. For instance, all objects know the \char`\"{}intensity\char`\"{} property. If data-\/$>$intensity != NULL, the instensity is computed and stored in $\ast$data-\/$>$intensity.

If data is non-\/\+NULL and only in this case, \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic_aa6f20e6d108357f6355493c4fdd5bb2f}{process\+Hit\+Quantities()}} will also call ph-\/$>$transmit() to update the transmissions of the \mbox{\hyperlink{class_gyoto_1_1_photon}{Photon}} (see \mbox{\hyperlink{class_gyoto_1_1_photon_a90545a8f03ab57524021baf0e19a61d3}{Photon\+::transmit(size\+\_\+t, double)}}). \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_disk3_d_a88f58b8244ee5ec4cdade4ba38cba23b}{Impact()}} may not extend the ph \mbox{\hyperlink{class_gyoto_1_1_worldline}{Worldline}}. The only two dates that are guaranteed to be defined are at indices index and index+1.

Whether this class is thread-\/safe. 

Return True if this object is thread-\/safe, i.\+e. if an instance and its clone can be used in parallel threads (in the context of Scenery\+::raytrace()). Known objects which are not thread-\/safe include Lorene metrics and everything from the \mbox{\hyperlink{namespace_gyoto_1_1_astrobj_1_1_python}{Python}} plug-\/in.

The default implementation considers that the class itself is thread safe and recurses into the declared properties to check whether they are safe too. Classes that abide to the Object/\+Property paradigm and are themselves thread-\/safe have nothing special to do.

Objects that clone children in their copy constructor that are not declared as properties must take these children into account.

Classes that are never thread-\/safe must declare it. It acn be easily done using GYOTO\+\_\+\+OBJECT\+\_\+\+THREAD\+\_\+\+SAFETY in the class declaration and GYOTO\+\_\+\+PROPERTY\+\_\+\+THREAD\+\_\+\+UNSAFE in the class definition. \mbox{\Hypertarget{class_gyoto_1_1_object_aae3b80667884c6620c4796a1b8ef74e5}\label{class_gyoto_1_1_object_aae3b80667884c6620c4796a1b8ef74e5}}
