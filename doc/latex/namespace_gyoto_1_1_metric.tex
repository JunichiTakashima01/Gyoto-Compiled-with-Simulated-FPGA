\doxysubsection{Detailed Description}
Access to metrics. 

Objects which describe space-\/time geometry must inherit from the \mbox{\hyperlink{class_gyoto_1_1_metric_1_1_generic}{Gyoto\+::\+Metric\+::\+Generic}} class.

To be usable, a \mbox{\hyperlink{class_gyoto_1_1_metric_1_1_generic}{Metric\+::\+Generic}} sub-\/class should register a \mbox{\hyperlink{namespace_gyoto_1_1_metric_ad6b33a642cc94ad67855184d2adb95df}{Metric\+::\+Subcontractor\+\_\+t}} function using the \mbox{\hyperlink{namespace_gyoto_1_1_metric_a602cba49b001e7470f3d439d1ab3e8f7}{Metric\+::\+Register()}} function. See also \mbox{\hyperlink{writing_plugins_page}{Writing plug-\/ins for Gyoto}} .

A function to build instances of a specific \mbox{\hyperlink{class_gyoto_1_1_metric_1_1_generic}{Metric\+::\+Generic}} sub-\/class. 

This is a more specific version of the \mbox{\hyperlink{class_gyoto_1_1_smart_pointee_afe17bea3c33e2187940cfdd5543c1d14}{Smart\+Pointee\+::\+Subcontractor\+\_\+t}} type. A Metric\+::\+Subcontrator\+\_\+t is called by the \mbox{\hyperlink{class_gyoto_1_1_factory}{Gyoto\+::\+Factory}} to build an instance of the kind of metric specified in an XML file (see \mbox{\hyperlink{namespace_gyoto_1_1_metric_a602cba49b001e7470f3d439d1ab3e8f7}{Register()}}). The \mbox{\hyperlink{class_gyoto_1_1_factory}{Factory}} and Subcontractor\+\_\+t function communicate through a \mbox{\hyperlink{class_gyoto_1_1_factory_messenger}{Gyoto\+::\+Factory\+Messenger}}.

Query the \mbox{\hyperlink{namespace_gyoto_1_1_metric}{Metric}} register. 

Query the \mbox{\hyperlink{namespace_gyoto_1_1_metric}{Metric}} register to get the \mbox{\hyperlink{namespace_gyoto_1_1_metric_ad6b33a642cc94ad67855184d2adb95df}{Metric\+::\+Subcontractor\+\_\+t}} correspondig to a given kind name. This function is normally called only from the \mbox{\hyperlink{class_gyoto_1_1_factory}{Factory}}. If plugin is specified, only a subcontractor matching both name and plugin will be returned, loading the plug-\/in if necessary. If plugin is the empty string, then the first subcontractor matching name will be returned, and the name of the plug-\/in it belongs to will be returned in plugin upon output.
