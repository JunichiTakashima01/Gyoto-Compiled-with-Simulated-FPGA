\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_gyoto_1_1_property}{Property}} that can be set and got using standard methods. 

The \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} API makes it easy to declare the parameters that can be set in a class.

Developpers who simply write classes (deriving from \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic}{Astrobj\+::\+Generic}}, , \mbox{\hyperlink{class_gyoto_1_1_metric_1_1_generic}{Metric\+::\+Generic}}, \mbox{\hyperlink{class_gyoto_1_1_spectrum_1_1_generic}{Spectrum\+::\+Generic}}) need not know the inners of the \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} class and interact with it only using macros to declare the parameters they need to read from XML.

To make use of the \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} framework, a class must derive from \mbox{\hyperlink{class_gyoto_1_1_object}{Gyoto\+::\+Object}} and use the \mbox{\hyperlink{_gyoto_object_8h_a750e7a43779f63bdb13111eaf0271203}{GYOTO\+\_\+\+OBJECT}} in a public section of the class declaration (i.\+e. in the .h file). Then, in the corresponding .C file, the GYOTO\+\_\+\+PROPERTY\+\_\+$\ast$ macros are used as follows (note the absence of punctuation after the macros)\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{_gyoto_property_8h_add8e16ed1e17ab2683b08cc330b341d4}{GYOTO\_PROPERTY\_START}}(MyClass)}
\DoxyCodeLine{GYOTO\_PROPERTY\_<type>(MyClass, PropertyName, accessor)}
\DoxyCodeLine{...}
\DoxyCodeLine{GYOTO\_PROPERTY\_END(MyClass, ParentClass::properties)}

\end{DoxyCode}


In the above, \mbox{\hyperlink{_gyoto_property_8h_add8e16ed1e17ab2683b08cc330b341d4}{GYOTO\+\_\+\+PROPERTY\+\_\+\+START}} starts the definition of the static member My\+Class\+::properties. Each GYOTO\+\_\+\+PROPERTY\+\_\+$<$type$>$ macro declares a new property. \mbox{\hyperlink{_gyoto_property_8h_a1356435ea7ee0e18d07201a80e034dcc}{GYOTO\+\_\+\+PROPERTY\+\_\+\+END}} ends the definition of the property list, with an optional pointer to the parent\textquotesingle{}s class \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} list, and defines the My\+Class\+::get\+Properties() method.

The underlying accessors must always be defined, both to set and to get the property. For the sake of simplicity, only a limited number of data types are allowed\+:
\begin{DoxyItemize}
\item double\+: see \mbox{\hyperlink{_gyoto_property_8h_a51b9b6d83536618f8c5c04101168305e}{GYOTO\+\_\+\+PROPERTY\+\_\+\+DOUBLE}}, \mbox{\hyperlink{_gyoto_property_8h_a10d55318bfccf207220f2adbcd0f00f9}{GYOTO\+\_\+\+PROPERTY\+\_\+\+DOUBLE\+\_\+\+UNIT}};
\item long\+: see \mbox{\hyperlink{_gyoto_property_8h_a1d76d0c83fdfbde84b66e5a3bbaa35f6}{GYOTO\+\_\+\+PROPERTY\+\_\+\+LONG}};
\item unsigned long\+: see \mbox{\hyperlink{_gyoto_property_8h_aca1f21d44168f8c9d0c30644b3785e9e}{GYOTO\+\_\+\+PROPERTY\+\_\+\+UNSIGNED\+\_\+\+LONG}} (a.\+k.\+a. size\+\_\+t\+: see \mbox{\hyperlink{_gyoto_property_8h_a63fbef357d49b36a1b91b4e111505e71}{GYOTO\+\_\+\+PROPERTY\+\_\+\+SIZE\+\_\+T}}, this may break on architectures where size\+\_\+t is not the same as unsigned long);
\item bool\+: see \mbox{\hyperlink{_gyoto_property_8h_ac9d7d689f6dc645598a81ec4505566df}{GYOTO\+\_\+\+PROPERTY\+\_\+\+BOOL}};
\item std\+::vector$<$double$>$\+: see \mbox{\hyperlink{_gyoto_property_8h_a93dda6fe102cce2ab1a212919673471c}{GYOTO\+\_\+\+PROPERTY\+\_\+\+VECTOR\+\_\+\+DOUBLE}} and \mbox{\hyperlink{_gyoto_property_8h_afee341ef4d305f0c2ab3c502bf2f11a5}{GYOTO\+\_\+\+PROPERTY\+\_\+\+VECTOR\+\_\+\+DOUBLE\+\_\+\+UNIT}};
\item std\+::vector$<$unsigned long$>$\+: see \mbox{\hyperlink{_gyoto_property_8h_a250f06ca284f2051db94fb2261e015bd}{GYOTO\+\_\+\+PROPERTY\+\_\+\+VECTOR\+\_\+\+UNSIGNED\+\_\+\+LONG}};
\item Gyoto\+::\+Smart\+Pointers to various base classes\+: \mbox{\hyperlink{class_gyoto_1_1_screen}{Screen}}, \mbox{\hyperlink{class_gyoto_1_1_metric_1_1_generic}{Metric\+::\+Generic}}, \mbox{\hyperlink{class_gyoto_1_1_astrobj_1_1_generic}{Astrobj\+::\+Generic}}, \mbox{\hyperlink{class_gyoto_1_1_spectrum_1_1_generic}{Spectrum\+::\+Generic}} and \mbox{\hyperlink{class_gyoto_1_1_spectrometer_1_1_generic}{Spectrometer\+::\+Generic}}. See \mbox{\hyperlink{_gyoto_property_8h_a37725a62fbd22ad1c7cc9f8761262774}{GYOTO\+\_\+\+PROPERTY\+\_\+\+METRIC}}, \mbox{\hyperlink{_gyoto_property_8h_a678522f21614980995c53d49c314a48c}{GYOTO\+\_\+\+PROPERTY\+\_\+\+SCREEN}}, \mbox{\hyperlink{_gyoto_property_8h_a13b6886c3edc42335c40fe44b070184c}{GYOTO\+\_\+\+PROPERTY\+\_\+\+ASTROBJ}}, \mbox{\hyperlink{_gyoto_property_8h_a3e144c41ba2b37be4f981f0aedee7480}{GYOTO\+\_\+\+PROPERTY\+\_\+\+SPECTRUM}} and \mbox{\hyperlink{_gyoto_property_8h_a9dbed4301797442973a9c6730de3a3a7}{GYOTO\+\_\+\+PROPERTY\+\_\+\+SPECTROMETER}}.
\end{DoxyItemize}

For the floating point data-\/types (double and vector$<$double$>$), two additional accessors supporting units can be provided. The accessors must have the same name and have specific prototypes, see the various function pointer typedefs, e.\+g. \mbox{\hyperlink{class_gyoto_1_1_property_a5c3fdfa4811f9eaf9fe5828d0d815597}{set\+\_\+double\+\_\+t}} and \mbox{\hyperlink{class_gyoto_1_1_property_ad854709d822b4632f7bd064186dd5fb1}{get\+\_\+double\+\_\+t}}.

The type used in these accessors may not be the same as the type of the underlying class member. For instance, to read an array, it was chosen to use the std\+::vector$<$type$>$ type because it is easy to read such a vector from XML and to thus determine dynamically the number of elements provided. But this type is slow, so it is expected that the class member will rather be a C-\/style array (double arr\mbox{[}\mbox{]}) or something else entirely. It is not forbidden to have a set of high-\/level accessors for the \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} interface on top of lower-\/level, more efficient accessors to be used in compiled, static code\+:

\begin{DoxyCode}{0}
