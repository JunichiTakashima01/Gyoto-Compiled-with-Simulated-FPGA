\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_gyoto_1_1_object}{Object}} with properties. 

The \mbox{\hyperlink{class_gyoto_1_1_object}{Object}} API allows declaring a list of Properties that can be set and retrieved using a common, text-\/based interface. This interface simplifies a lot how to read and write XML, as well as writing bindings for interpreted langages (e.\+g. the Yorick interface).

In fact, any class member that has an interface implemented as a \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} can be readily read and written from/to XML as well as from the Yorick plug-\/in, without the need for any additional code.

To declare a \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} list\+:
\begin{DoxyEnumerate}
\item declare (in the class declaration, .h file) and define (.C file) the pair or quadruplet of accessors for your \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} (see \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} class documentation;
\item call the \mbox{\hyperlink{_gyoto_object_8h_a750e7a43779f63bdb13111eaf0271203}{GYOTO\+\_\+\+OBJECT}} macro in in a public section of the class declaration (in the .h file)\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{class }A: \textcolor{keyword}{public} \mbox{\hyperlink{class_gyoto_1_1_object_a5834e83ca1482a379c7ddefb1919457f}{Object}} \{}
\DoxyCodeLine{  \textcolor{keyword}{public}:}
\DoxyCodeLine{    \mbox{\hyperlink{_gyoto_object_8h_a750e7a43779f63bdb13111eaf0271203}{GYOTO\_OBJECT}};}
\DoxyCodeLine{    A();}
\DoxyCodeLine{    \textcolor{keyword}{virtual} \string~A();}
\DoxyCodeLine{  ...}
\DoxyCodeLine{\};}

\end{DoxyCode}

\item call the various GYOTO\+\_\+\+PROPERTY\+\_\+$\ast$ macros in the corresponding .C file (see the documentation of the \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} class).
\end{DoxyEnumerate}

It is possible to get a \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} by name (Assume {\itshape A} is a class deriving from \mbox{\hyperlink{class_gyoto_1_1_object}{Object}})\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{A myobj();}
\DoxyCodeLine{Property \textcolor{keyword}{const} *prop=NULL;}

\end{DoxyCode}
 It then becomes possible to set or get the \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} from or to a \mbox{\hyperlink{class_gyoto_1_1_value}{Value}}\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{myobj.set(*prop, \textcolor{keywordtype}{size\_t}(12));}

\end{DoxyCode}
 Of course the type of the \mbox{\hyperlink{class_gyoto_1_1_value}{Value}} instance and of the \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} instance must match. Refer to the documentation of these to classes for details.

Fill the XML element for this \mbox{\hyperlink{class_gyoto_1_1_object}{Object}}. 

The base implementation simply calls \mbox{\hyperlink{class_gyoto_1_1_object_a11892ef308d6f40add51eaea653a394b}{fill\+Property()}} for each \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} defined for the \mbox{\hyperlink{class_gyoto_1_1_object}{Object}}.

Derived classes should avoid overriding \mbox{\hyperlink{class_gyoto_1_1_object_a0ffc3f84b4c2372d639eb848605fb1f3}{fill\+Element()}}. It may make sense occasionally, e.\+g. to make sure that the metric is output first.

To customize how a given \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} is rendered, it is better to override \mbox{\hyperlink{class_gyoto_1_1_object_a11892ef308d6f40add51eaea653a394b}{fill\+Property()}}.

If this method is overridden, the implementation should in general call \mbox{\hyperlink{class_gyoto_1_1_object_a0ffc3f84b4c2372d639eb848605fb1f3}{fill\+Element()}} on the direct base.

Whether this class is thread-\/safe. 

Return True if this object is thread-\/safe, i.\+e. if an instance and its clone can be used in parallel threads (in the context of Scenery\+::raytrace()). Known objects which are not thread-\/safe include Lorene metrics and everything from the \mbox{\hyperlink{namespace_gyoto_1_1_python}{Python}} plug-\/in.

The default implementation considers that the class itself is thread safe and recurses into the declared properties to check whether they are safe too. Classes that abide to the Object/\+Property paradigm and are themselves thread-\/safe have nothing special to do.

Objects that clone children in their copy constructor that are not declared as properties must take these children into account.

Classes that are never thread-\/safe must declare it. It acn be easily done using GYOTO\+\_\+\+OBJECT\+\_\+\+THREAD\+\_\+\+SAFETY in the class declaration and GYOTO\+\_\+\+PROPERTY\+\_\+\+THREAD\+\_\+\+UNSAFE in the class definition. \mbox{\Hypertarget{class_gyoto_1_1_object_aae3b80667884c6620c4796a1b8ef74e5}\label{class_gyoto_1_1_object_aae3b80667884c6620c4796a1b8ef74e5}}
