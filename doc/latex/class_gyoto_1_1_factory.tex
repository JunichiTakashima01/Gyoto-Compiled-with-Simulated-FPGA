\doxysubsection{Detailed Description}
XML input/output. 

The \mbox{\hyperlink{class_gyoto_1_1_factory}{Factory}} is responsible from building objects from their XML description, and from saving an XML description of existing objects. Since the \mbox{\hyperlink{class_gyoto_1_1_factory}{Factory}} doesn\textquotesingle{}t know how to build the variety of objects available in \mbox{\hyperlink{namespace_gyoto}{Gyoto}} and in external plug-\/ins, the \mbox{\hyperlink{class_gyoto_1_1_factory}{Factory}} orders \mbox{\hyperlink{namespace_gyoto_1_1_metric}{Metric}}, \mbox{\hyperlink{namespace_gyoto_1_1_astrobj}{Astrobj}} and \mbox{\hyperlink{namespace_gyoto_1_1_spectrum}{Spectrum}} objects from registered subcontractors (see \mbox{\hyperlink{class_gyoto_1_1_smart_pointee_afe17bea3c33e2187940cfdd5543c1d14}{Smart\+Pointee\+::\+Subcontractor\+\_\+t}}). The factory an the various subcontractors communicate through a \mbox{\hyperlink{class_gyoto_1_1_factory_messenger}{Factory\+Messenger}}.

If it starts with \char`\"{}/\char`\"{}, it is interpreted as an absolute path and is returned as is.

If it is prefixed with \char`\"{}\`{}pwd\`{}/\char`\"{}, the rest of relpath is interpreted as relative to the current working directory, i.\+e. \mbox{\hyperlink{class_gyoto_1_1_factory_a1adf0078f2541926fcfeac0221a21206}{full\+Path()}} tries to mimic how the shell would expand relpath into an absolute path.
