\doxysubsection{Detailed Description}
Container for the value of a \mbox{\hyperlink{class_gyoto_1_1_property}{Property}}. 

The \mbox{\hyperlink{class_gyoto_1_1_value}{Value}} class is very similar to the C union type (although not as memory efficient)\+: it can hold several type of values, but only one at a time. Care must be taken to ensure only the member that was set is retrieved. The purpose of the \mbox{\hyperlink{class_gyoto_1_1_value}{Value}} class is to be used together with the \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} class\+: code determines dynamicaly the type of a \mbox{\hyperlink{class_gyoto_1_1_property}{Property}}, reads the corresponding value appropriateley (e.\+g. from XML or from the Yorick prompt), stores the value in a \mbox{\hyperlink{class_gyoto_1_1_value}{Value}} instance, and sets the \mbox{\hyperlink{class_gyoto_1_1_property}{Property}} using the \mbox{\hyperlink{class_gyoto_1_1_object_a58c4ecdaae1873dc6912e164b61826e0}{Object\+::set()}} method. Likewise, the \mbox{\hyperlink{class_gyoto_1_1_object_a8232112e660216a61caa15fab27b889f}{Object\+::get()}} method returns a \mbox{\hyperlink{class_gyoto_1_1_value}{Gyoto\+::\+Value}}. \mbox{\hyperlink{class_gyoto_1_1_property_affac9b81023457fb24781c7dd678f154}{Property\+::type}} must be used to determine which member of the \mbox{\hyperlink{class_gyoto_1_1_value}{Value}} is meaningful.

Casting between \mbox{\hyperlink{class_gyoto_1_1_value}{Value}} and the various data type it can hold is normally automatic, but the members can also be accessed explicitly make code more easy to read and less ambiguous.
